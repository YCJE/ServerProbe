## 1. 安全审计清单

开发完成后，按此清单逐项审计。所有项目必须全部通过方可发布。

### 1.1 核心安全原则审计（S1-S8）

| 编号 | 审计项 | 验证方法 | 通过标准 | 状态 |
|------|--------|---------|---------|------|
| S1-1 | 协议中无控制帧 | 审查 WebSocket 消息协议定义 | Server→Agent 方向只有 register_ok/register_fail/config_update/heartbeat_ack 四种帧 | [ ] |
| S1-2 | Server 不主动连接 Agent | 全局搜索 `net.Dial`（Server 主动连接 Agent 的方向） | 零匹配 | [ ] |
| S4-1 | 无命令执行 | 全局搜索 `os/exec`、`exec.Command`、`exec.CommandContext` | Agent 侧零匹配 | [ ] |
| S4-2 | 无终端会话 | 全局搜索 `pty`、`term`、`shell`、`terminal` | Agent 侧零匹配 | [ ] |
| S4-3 | 无文件系统操作 | 全局搜索 `os.OpenFile`（写模式）、`os.Remove`、`os.Rename` | Agent 侧零匹配（配置文件写入除外） | [ ] |
| S4-4 | 无 Cron 任务执行器 | 审查 Agent 代码结构 | 不包含 Cron 任务执行器 | [ ] |
| S2-1 | TLS 配置不可关闭 | 审查 Server 配置加载逻辑 | TLS 为强制项，无关闭选项 | [ ] |
| S2-2 | Agent 拒绝明文连接 | 审查 Agent 连接逻辑 | 拒绝 `http://` 和 `ws://`，仅接受 `https://` 和 `wss://` | [ ] |
| S2-3 | 无证书时自动生成自签证书 | 审查 Server 启动逻辑 | 无证书时自动生成自签证书并记录告警日志 | [ ] |

| S6-2 | DNS 重绑定防护 | 审查 HTTP Client 配置 | 自定义 Dialer 强制使用预解析 IP | [ ] |
| S6-3 | 重定向检测 | 测试重定向到内网 | CheckRedirect 中再次 SSRF 检查 | [ ] |
